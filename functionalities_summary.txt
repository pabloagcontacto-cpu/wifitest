WIFITEST - Resumen de funcionalidades propuestas
================================================

Este documento resume funcionalidades candidatas para el dashboard de WIFITEST.
Cada funcionalidad se expresa como una pieza de frontend que puede apoyarse en una
o varias tools MCP. La separacion principal se organiza por:

- Estado operativo: no conectado / conectado
- Tipo de funcionalidad: reconocimiento pasivo / reconocimiento activo / pruebas de seguridad / ataques activos

Notas:
- "No conectado" significa sin estar asociado a la red objetivo.
- "Conectado" significa una vez unido a la red objetivo o a la red que se quiere auditar.
- Las funcionalidades agresivas se incluyen como parte del mapa completo del proyecto,
  pero no implican recomendacion de exponerlas en un flujo normal de usuario.
